## SPI协议

**SPI（Serial Peripheral Interface，串行外设接口）** 是一种由摩托罗拉（Motorola）公司提出的**同步串行通信协议**，主要用于嵌入式系统中微控制器（MCU）与外围设备之间的短距离、高速数据传输。

> 🔑 核心特点：**高速、全双工、同步、主从架构**

---

## 🔌 SPI信号线定义（4线标准）

| 信号线       | 全称                         | 方向    | 功能说明                  |
| --------- | -------------------------- | ----- | --------------------- |
| **SCLK**  | Serial Clock               | 主机→从机 | 时钟信号，由主机产生，同步数据传输     |
| **MOSI**  | Master Out Slave In        | 主机→从机 | 主机发送数据，从机接收           |
| **MISO**  | Master In Slave Out        | 从机→主机 | 从机发送数据，主机接收           |
| **SS/CS** | Slave Select / Chip Select | 主机→从机 | 片选信号，低电平有效，用于选择通信的从设备 |

```
        ┌─────────────┐         ┌─────────────┐
        │   Master    │         │    Slave    │
        │   (MCU)     │         │  (Peripheral)│
        ├─────────────┤         ├─────────────┤
   SCLK │────────────▶│         │             │
   MOSI │────────────▶│         │◀────────────│ MISO
   MISO │◀────────────│         │────────────▶│ MOSI
    SS  │────────────▶│         │             │
        └─────────────┘         └─────────────┘
```

## ⚙️ 工作原理

### 1️⃣ 主从架构
- **主机（Master）**：负责产生时钟信号（SCLK）和片选信号（SS），控制通信流程
- **从机（Slave）**：响应主机的片选信号，在时钟驱动下进行数据收发

### 2️⃣ 通信流程
1. 主机将目标从机的 **SS信号拉低**，选中该从机
2. 主机产生 **SCLK时钟信号**
3. 在每个时钟周期内：
   - 主机通过 **MOSI** 发送1位数据
   - 从机通过 **MISO** 发送1位数据
   - 双方**同时**读取对方发送的数据（全双工）
4. 通信结束，主机将 **SS拉高**，释放从机

### 3️⃣ 数据传输特点
- **同步传输**：数据在时钟边沿采样，无需起始/停止位
- **全双工**：发送和接收可同时进行
- **移位寄存器**：数据按位（bit）串行传输，通常8位或16位为一帧

## 🔄 SPI四种工作模式

由**时钟极性（CPOL）** 和 **时钟相位（CPHA）** 组合决定：

| 模式 | CPOL | CPHA | 时钟空闲电平 | 数据采样边沿 | 数据输出边沿 |
|------|------|------|-------------|-------------|-------------|
| **Mode 0** | 0 | 0 | 低电平 | 上升沿⬆️ | 下降沿⬇️ |
| **Mode 1** | 0 | 1 | 低电平 | 下降沿⬇️ | 上升沿⬆️ |
| **Mode 2** | 1 | 0 | 高电平 | 下降沿⬇️ | 上升沿⬆️ |
| **Mode 3** | 1 | 1 | 高电平 | 上升沿⬆️ | 下降沿⬇️ |

> 📝 **关键参数解释**：
> - **CPOL（Clock Polarity）**：时钟空闲时的电平状态
> - **CPHA（Clock Phase）**：数据在哪个时钟边沿被采样

⚠️ **注意**：通信双方必须配置**相同的模式**，否则数据会错乱！

## 🎯 典型应用场景

| 应用领域     | 具体设备                |
| -------- | ------------------- |
| 🗄️ 存储设备 | EEPROM、Flash、SD卡    |
| 🌡️ 传感器  | 温湿度、压力、加速度计、ADC/DAC |
| 🖥️ 显示设备 | LCD屏幕、OLED、触摸屏      |
| 🔊 音频设备  | 音频编解码器（Codec）、数字电位器 |
| ⏰ 实时时钟   | RTC模块               |
| 🔧 工业控制  | PLC、数字信号处理器、电机驱动    |

## ✅ SPI优缺点对比

### ✔️ 优点
- **高速传输**：速率可达几十Mbps，远高于I2C
- **全双工通信**：收发同时进行，效率高
- **协议简单**：无复杂地址/应答机制，易于实现
- **灵活配置**：支持4种模式，适配不同外设

### ❌ 缺点
- **占用引脚多**：每增加一个从机需额外片选线
- **无应答机制**：无法确认从机是否成功接收
- **距离受限**：通常仅适用于板级短距离通信（<1米）
- **无标准协议**：不同厂商实现可能有差异

## 🔧 与I2C协议对比

| 特性 | SPI | I2C |
|------|-----|-----|
| 信号线 | 4根（SCLK/MOSI/MISO/SS） | 2根（SDA/SCL） |
| 通信方式 | 全双工 | 半双工 |
| 传输速率 | 高（10-50+ Mbps） | 中（100k-3.4 Mbps） |
| 从机寻址 | 通过片选线硬件选择 | 通过地址软件寻址 |
| 协议复杂度 | 简单 | 较复杂（含起始/停止/应答） |
| 适用场景 | 高速、实时性要求高 | 引脚紧张、多设备挂载 |
